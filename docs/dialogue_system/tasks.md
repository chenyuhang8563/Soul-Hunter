# Dialogue System Tasks

## 阶段 1：基础 UI 搭建 (UI Setup)
- [ ] 创建 `DialogueUI.tscn` 场景作为基础 UI 载体。
- [ ] 配置主对话框背景（PanelContainer），设置合适的样式与布局（位于屏幕底部）。
- [ ] 在对话框内添加头像区域（TextureRect）、名称文本（Label）和对话主文本（RichTextLabel）。
- [ ] 添加右下角的“下一句”指示箭头，并为其制作简单的浮动动画（Tween 或 AnimationPlayer）。
- [ ] 在对话框右上角添加选项容器（VBoxContainer）及选项按钮预制件。

## 阶段 2：对话数据结构与管理器 (Data & Manager)
- [ ] 编写 `dialogue_ui.gd` 脚本控制界面内聚逻辑。
- [ ] 设计对话数据字典格式（节点化设计，支持 `next_id` 跳转）。
- [ ] 创建全局的 `DialogueManager.gd` 自动加载单例，用于随时随地呼出对话框并管理生命周期。

## 阶段 3：交互逻辑实现 (Interaction)
- [ ] 实现打字机效果（通过代码逐字显示对话内容）。
- [ ] 实现 `Space` / 鼠标点击 推进对话的逻辑（打字中则补全，打字完且无选项则进入下一句）。
- [ ] 实现选项的动态生成逻辑。
- [ ] 绑定选项的鼠标点击事件，以及对应的键盘数字键（1、2等）快捷键。
- [ ] 处理对话结束逻辑（隐藏 UI，清理状态数据，发送 `dialogue_finished` 信号）。

## 阶段 4：测试与调整 (Testing)
- [ ] 编写一段测试对话数据（包含单句、连续对话、分支选项）。
- [ ] 在测试场景中调用并验证鼠标点击、空格键推进表现是否顺畅。
- [ ] 验证数字键和点击选项的正确跳转，确保无卡死或空指针。
- [ ] 确认对话框的层级正常，不被其他 UI 遮挡，且能正确拦截底层的游戏交互。
